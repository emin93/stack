fresh mac setup

install.emin.ch

A single command turns a freshly unboxed Apple Silicon Mac into a working dev environment. Safe to re-run on any existing setup.

~/ — zsh
$ bash -c "$(curl -fsSL https://install.emin.ch/install.sh)"
Apple Silicon Idempotent ~30 min unattended No npm-pipe-bash sketchiness

What you get

CLI tools
ripgrep fd gh stow ffmpeg xcodes aria2
Apps
Ghostty · 1Password · Brave · Claude Code · Codex · Android Studio
Runtimes
node, pnpm, and go via Homebrew — latest stable, one global version each
Dotfiles
zsh, git, and Ghostty configs symlinked from ~/Documents/Projects/install via stow
Auth
GitHub (gh + new ed25519 SSH key, uploaded for you), Claude, Codex, Apple ID for Xcode — all walked through interactively
Xcode
Command Line Tools + latest Xcode, installed via xcodes (the slow one — run it overnight)

Inside the script

  1. Sanity check. macOS, Apple Silicon, not root, not truncated.
  2. Xcode Command Line Tools. Triggers the GUI installer if missing.
  3. Homebrew. Installs and wires up brew shellenv.
  4. Clone the repo. HTTPS, so it works before any auth is set up.
  5. Brew bundle. Installs/updates every tool and cask declared in the Brewfile.
  6. GitHub auth. Device-code login plus gh auth setup-git as credential helper.
  7. SSH key. Generates ed25519 and uploads to your GitHub account.
  8. Stow. Symlinks zsh, git, and Ghostty configs into $HOME, backing up existing files.
  9. Override stubs. Creates empty .gitconfig.local, .zshrc.local, config.local.
  10. Claude + Codex sign-in. Launches each CLI for interactive auth, then continues.
  11. Xcode. The long pole — xcodes install --latest --select.

What you still do by hand

Two things the script can't do for you:
  • Fill in ~/.gitconfig.local with your name and email
  • Sign in to apps with GUI-only auth (1Password, Brave sync)
